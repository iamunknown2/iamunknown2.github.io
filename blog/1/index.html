An Introduction to JavaScript - Part 1

Introduction

If you're reading this, you're probably trying to learn how to program. Despite what some teachers will tell you, learning Pascal/Fortran/any language you've never heard of is not the way to learn programming nowadays, as Pascal was made between 1968 and 1969 - Yep, it was made when the military was the only group that had any access to ARPANET (the predecessor of the Internet)!

Nowadays, it's different. The web has its own industry (e.g Google). Guess what the programming language of the web is? HTML and CSS? Despite what you may believe, HTML and CSS isn't actually a programming language - HTML is just for displaying content, while CSS is to make the content prettier. Instead, the programming (or I should say scripting) language of the web is JavaScript.

As the title suggests, I'm going to teach you a bit of JavaScript. Let's start with the basics...


Before we begin...

Let's get started!


Alerting, logging, errors and all that stuff

For a start, see what happens when you type this into the console (that you presumably have open) before hitting Enter:

console.log("Hello world");

Wow! How does that work?

To explain how this works completely would require us to have a more advanced understanding of JS which will be unveiled later in this course. However, the best way to put it is that console.log() is a function (a command) that will print (aka log anything between the brackets (in this case, "Hello world") to the console.

What about those pop-up boxes? Can you make them in JavaScript?

Yes. In fact, you're supposed to make those pop-up boxes in JavaScript! Type this into the console before you hit Enter:

alert("Hello world");

As you can see, alert() is a function that outputs whatever's in the brackets (in this case, "Hello world") to the console.

I was talking about the kind of pop-up box that has 2 options!

Fine. It's

confirm("This is a confirm message.")

And while you're at it, see what this does:

prompt("Input something:")

But in order to make the confirm/prompt boxes interactive, it'll require us to have at least a basic understanding of variables/arrays etc, which I will cover in the next post.
